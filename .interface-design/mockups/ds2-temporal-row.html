Cairn states — reference
Cairn — three states
Cairns follow the same present/adjacent language as the disagreement system, with a different visual register.
Present (filled △) — note at your exact context.
Adjacent (hollow △) — note nearby but not aligned.
Active (filled ▽) — selected, row expands to show content.

Pip states — reference
Pips — filled / hollow (active only)
Same visual rule as cairns: filled = present, hollow = adjacent.
Present (filled ●) — active workflow item at your exact context.
Adjacent (hollow ○) — active workflow item nearby, not aligned.
Color carries the spectrum: pencil (change), redline (conflict), overlay (directive).
No ghost/resolved state. When a workflow item resolves, its pip disappears. Cairns mark resolved-but-pending stories. Full silence when lifecycle completes.
