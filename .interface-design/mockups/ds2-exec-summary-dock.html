Design notes
Design decisions captured in this mockup (v1)
1. Two zones, one panel. The right panel contains two zones: workflow tree (top, flex priority) and notes area (bottom, up to 40%). Each scrolls independently. The tree shows project-wide workflow state; the notes area shows context for the current item.

2. Workflow tree = exec summary. Three-level navigable tree: category (Conflicts, Changes, Directives, Resolved, Notes) → type (Doors, Hardware) → instance (Door 101 · fire rating). Category borders use spectrum colors. Clicking an instance triggers a page turn to the main area.

3. Notes tree vs. notes area. The "Notes" category in the workflow tree is a global rollup — all cairns across the project. The notes area at the bottom is contextual — notes for the one item currently in the main panel. Both reference the same cairn items in the graph.

4. Notes area tracks the main panel. When you navigate to Door 101, its notes appear. When you page-turn to a conflict item, those notes appear. At project/dashboard level, the empty state shows. The bottom zone follows the story panel's focus.

5. Auto-expand/collapse. The notes area auto-expands when navigating to an item that has notes. It collapses by default when the current item has no notes. The user can manually toggle either way.

6. Kernel badge. When collapsed, the panel shows a badge with the count of open workflow items. Redline badge if items need review. Stamp badge (checkmark) when everything is resolved. The user always knows the project state.

7. Active instance highlight. The currently-viewed item gets a subtle highlight (sheet background) in the workflow tree, so you can see where you are in the project context.

8. Status in tree. In-review and hold status show as small inline badges on instances. This makes the exec summary the primary status management surface — you can see who's reviewing what without leaving the dock.
